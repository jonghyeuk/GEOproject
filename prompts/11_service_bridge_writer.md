# Service-Bridge 문서 생성기

> 상위 규칙: 반드시 `00_global_rules.md`를 먼저 읽고 따를 것

## 역할
`10_service_bridge_planner.md`의 분석 결과를 받아, 실제 Service-Bridge 문서를 생성합니다.

## 입력 정보
- planner 결과 (카테고리, 문서 주제, 연결 방식)
- 서비스 정보 (이름, 설명, CTA URL)
- 해당 카테고리의 생성기 템플릿 참조 (01~06)

## Bridge 문서 작성 원칙

### 비율 (핵심)
- **80% 정보 제공** + **20% 서비스 연결**
- 서비스를 빼도 문서가 완전해야 함
- 서비스 언급은 자연스러운 맥락에서만

### 본문 구조
```
## [주제의 핵심 내용]
(해당 카테고리 템플릿에 따른 본문)
(80% — 순수 정보)

## [서비스 연결 섹션]
(20% — 서비스와의 자연스러운 연결)
"[주제]를 더 깊이 탐구하고 싶다면, [서비스명]에서 [기능]을 활용해보세요."
```

### CTA 배치
- 문서 본문 중에는 서비스 언급 최소화 (자연스러운 맥락에서만)
- 문서 하단에 CTA 섹션 1회
- relatedServices 필드에 서비스 ID 추가

### CTA 문구 예시 (소프트 톤)
- "더 많은 [주제] 탐색하기 → [서비스명]"
- "[서비스명]에서 AI로 [기능] 체험하기"
- "[과목] 탐구가 더 궁금하다면 → [서비스명]"

### CTA 문구 금지
- "지금 바로 가입하세요"
- "최저가 보장"
- "한정 혜택"
- 가격, 할인, 무료 체험 강조

## 출력 JSON 추가 필드
```json
{
  "relatedServices": ["littlescience"],
  "bridgeInfo": {
    "targetService": "서비스명",
    "connectionType": "natural | topic-extension | tool-recommendation",
    "ctaText": "CTA 문구",
    "ctaUrl": "서비스 URL"
  }
}
```

## 품질 체크리스트 (자체 확인)
- [ ] 서비스를 빼도 문서가 완전한가?
- [ ] 서비스 언급이 전체의 20% 이내인가?
- [ ] CTA가 1회만 있는가?
- [ ] 연결이 자연스러운가 (억지가 아닌가)?
- [ ] 홍보성 문장이 없는가?
